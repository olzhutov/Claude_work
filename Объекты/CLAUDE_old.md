# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

---

## Роль и назначение

Этот репозиторий — **вики по коммерческой недвижимости**, которую поддерживает Claude. Задача Claude — работать как «компилятор»: читать сырые документы из `/raw`, извлекать факты, создавать и обновлять структурированные страницы вики, поддерживать перекрёстные ссылки и разрешать противоречия. Пользователь никогда не пишет вики напрямую — только добавляет сырые источники и задаёт вопросы.

---

## Структура директорий

```
/raw/             ← сырые источники (неизменяемые). Только сюда кладёт файлы пользователь.
  /assets/        ← локальные копии изображений и вложений

/wiki/
  /objects/       ← одна страница на каждый объект недвижимости
  /tenants/       ← одна страница на каждого арендатора
  /contracts/     ← одна страница на каждый договор аренды
  /financials/    ← сводные финансовые анализы (NOI, доходность, сценарии)
  /topics/        ← концептуальные страницы (рыночные ставки, методология оценки и др.)

index.md          ← каталог всех страниц вики (обновляется при каждом ингесте)
log.md            ← хронологический лог всех операций (только дописывается)
```

### Финансовый анализ (`/wiki/financials/`)

Имя файла: `НазваниеОбъекта_финансы_ГГГГ.md` или `НазваниеОбъекта_сценарии.md`.

Свободный формат, но обязательно: все цифры со ссылками на источники (`[[raw/...]]` или `[[contracts/...]]`), явная дата расчёта и допущения.

---

## Правила работы с источниками

1. **Каждый факт** должен иметь ссылку на источник в формате `[[raw/имя_файла]]`. Если источников несколько — перечислить все.
2. **Источники неизменяемы.** Claude никогда не редактирует файлы в `/raw`.
3. **Дата документа** определяет актуальность. Если дата не указана явно — использовать дату файла или дату из содержимого; если неизвестна — пометить `[дата неизвестна]`.

---

## Разрешение противоречий

Когда два источника дают разные значения для одного факта (например, разные арендные ставки):

1. Определить, какой документ новее по дате.
2. Установить новое значение как «истину» на странице.
3. Старое значение не удалять сразу — добавить в раздел **«История изменений»** с пометкой `[устарело]` и ссылкой на оба источника.
4. Если дату определить невозможно — пометить конфликт тегом `[КОНФЛИКТ]` прямо в тексте и запросить у пользователя уточнение.

Формат записи конфликта:
```
> [КОНФЛИКТ] Ставка аренды: 25 $/м²/мес (→ [[raw/договор_2023.pdf]]) vs 28 $/м²/мес (→ [[raw/письмо_2024.docx]]). Требуется уточнение.
```

## Операции

### Ингест (`ingest`)

Когда пользователь добавляет новый файл в `/raw` и просит его обработать:

1. Прочитать документ, выделить ключевые факты.
2. Кратко обсудить с пользователем главные выводы (особенно неожиданные данные или противоречия).
3. Создать или обновить страницу **объекта**, **арендатора**, **договора** — в зависимости от содержимого.
4. Обновить связанные страницы (например, добавить арендатора на страницу объекта).
5. Обновить `index.md`.
6. Дописать запись в `log.md`: `## [YYYY-MM-DD] ingest | Имя файла | краткое описание`.

Один документ может затронуть 5–15 страниц вики.

### Запрос (`query`)

Когда пользователь задаёт аналитический вопрос:

1. Прочитать `index.md`, найти релевантные страницы.
2. Прочитать страницы, синтезировать ответ с цитатами.
3. **Если ответ ценный** (сравнение, анализ, вывод) — предложить сохранить его как новую страницу в `/wiki/financials/` или `/wiki/topics/`.
4. Дописать запись в `log.md`: `## [YYYY-MM-DD] query | Краткая формулировка вопроса`.

### Проверка (`lint`)

По запросу пользователя или самостоятельно раз в ~20 ингестов:

- Найти страницы без входящих ссылок (orphan pages).
- Найти факты без источников.
- Найти неразрешённые `[КОНФЛИКТ]`.
- Найти устаревшие данные (договоры с истёкшим сроком без обновления статуса).
- Предложить новые страницы для концептов, упомянутых несколько раз, но не имеющих своей страницы.

## log.md — формат

Только дописывается, никогда не редактируется. Каждая запись:

```
## [YYYY-MM-DD] тип_операции | Заголовок
Краткое описание: что сделано, какие страницы затронуты, какие противоречия найдены.
```

Типы операций: `ingest`, `query`, `lint`, `manual`.

## Технические правила

- **Язык вики**: русский (факты из украиноязычных документов переводить при необходимости, оригинальные термины оставлять в скобках).
- **Имена файлов**: без пробелов, кириллица допустима, нижнее подчёркивание как разделитель.
- **Валюта**: фиксировать ту, что в источнике; при пересчёте указывать курс и дату.
- **Площади**: всегда в м², всегда с источником.
- **Ссылки**: использовать формат `[[папка/имя_файла]]` без расширения `.md` — это стандарт Obsidian.
- **Не создавать** директории `/raw/` и `/assets/` — пользователь создаёт их сам.
- **Не изменять** `.obsidian/` и файлы конфигурации.
